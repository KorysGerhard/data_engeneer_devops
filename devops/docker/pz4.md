# Практическое задание

**Контейнеризация мини-проекта Vite с использованием Docker, Dockerfile, Dockerfile.dev и томов**

Необходимо создать небольшой фронтенд-проект через Vite и запустить его внутри контейнера Docker. В процессе работы нужно создать два Dockerfile (для разработки и для сборки приложения), использовать контейнеры Docker и подключить тома для работы с кодом проекта.

# Цель

Вы сможете:

- запускать контейнеры Docker
- создавать собственные Dockerfile
- использовать отдельный Dockerfile для разработки
- подключать Docker volumes или bind mounts
- запускать фронтенд-проект Vite внутри контейнера
- работать с контейнером в режиме разработки

# Задание (шаги)

## Шаг 1. Создать новый проект Vite

Создайте новый проект.

```bash
npm create vite@latest vite-docker-app
```

Выберите любой шаблон (например `vanilla` или `react`).

После создания перейдите в директорию проекта:

```bash
cd vite-docker-app
```

Установите зависимости:

```bash
npm install
```

Проверьте, что приложение запускается локально:

```bash
npm run dev
```

После проверки остановите сервер разработки.

# Шаг 2. Создать Dockerfile.dev (контейнер для разработки)

В корне проекта создайте файл:

```
Dockerfile.dev
```

Этот файл будет использоваться только для разработки.

Пример структуры:

```dockerfile
FROM node:18

WORKDIR /app

COPY package.json package-lock.json ./

RUN npm install

COPY . .

EXPOSE 5173

CMD ["npm", "run", "dev", "--", "--host"]
```

Разберём основные части.

`FROM node:18`
Используется базовый образ с Node.js.

`WORKDIR /app`
Задаётся рабочая директория контейнера.

`COPY package.json package-lock.json ./`
Копируются файлы зависимостей.

`RUN npm install`
Устанавливаются зависимости.

`COPY . .`
Копируется весь код проекта.

`EXPOSE 5173`
Порт, который использует Vite.

`CMD`
Команда запуска сервера разработки.

# Шаг 3. Собрать образ разработки

Соберите образ Docker.

```bash
docker build -f Dockerfile.dev -t vite-dev .
```

Здесь:

`-f Dockerfile.dev` — указание файла сборки
`-t vite-dev` — имя образа

Проверьте, что образ появился:

```bash
docker images
```

# Шаг 4. Запустить контейнер разработки

Теперь запустите контейнер.

```bash
docker run -p 5173:5173 vite-dev
```

Откройте браузер:

```
http://localhost:5173
```

Приложение должно работать внутри контейнера.

Остановите контейнер.

# Шаг 5. Использовать bind mount для разработки

Теперь нужно сделать так, чтобы контейнер автоматически видел изменения файлов проекта.

Запустите контейнер с bind mount:

```bash
docker run -p 5173:5173 -v $(pwd):/app vite-dev
```

Теперь:

- код редактируется на компьютере
- контейнер сразу видит изменения

Измените файл `App.jsx` или `main.js`.

Обновите страницу браузера.

Изменения должны появиться.

# Шаг 6. Создать production Dockerfile

Теперь создайте второй Dockerfile для production-версии приложения.

Создайте файл:

```
Dockerfile
```

Пример:

```dockerfile
FROM node:18 AS builder

WORKDIR /app

COPY package.json package-lock.json ./

RUN npm install

COPY . .

RUN npm run build


FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html

EXPOSE 80
```

Этот Dockerfile использует **multi-stage build**.

# Шаг 7. Собрать production-образ

Соберите образ:

```bash
docker build -t vite-prod .
```

# Шаг 8. Запустить production контейнер

Запустите контейнер.

```bash
docker run -p 8080:80 vite-prod
```

Откройте браузер:

```
http://localhost:8080
```

Должна открыться production-версия приложения.

# Подсказки по ключевым частям

### 1. Dockerfile.dev используется только для разработки

Он должен:

- запускать `npm run dev`
- открывать порт Vite
- поддерживать hot reload

Поэтому используется bind mount.

### 2. Production Dockerfile должен быть минимальным

В production контейнере не нужны:

- node_modules
- dev зависимости
- инструменты сборки

Поэтому используется multi-stage build.

### 3. Bind mount нужен для разработки

Команда:

```bash
-v $(pwd):/app
```

означает:

```
локальная папка проекта → папка /app внутри контейнера
```

### 4. Проверяйте порты

Vite использует порт:

```
5173
```

Поэтому важно пробрасывать порт:

```
-p 5173:5173
```

# Советы по улучшению работы

### 1. Добавьте `.dockerignore`

Чтобы не копировать лишние файлы.

Пример:

```
node_modules
.git
dist
Dockerfile
Dockerfile.dev
```

### 2. Используйте конкретные версии образов

Лучше писать:

```
node:18
```

чем:

```
node:latest
```

Это делает сборку стабильнее.

### 3. Разделяйте зависимости и код

В Dockerfile сначала копируйте `package.json`, затем выполняйте `npm install`, а потом копируйте код.

Это ускоряет сборку образов.

### 4. Проверяйте размер образа

Команда:

```bash
docker images
```

Позволяет увидеть размер образа.

### 5. Останавливайте контейнеры

После тестирования контейнеров не забывайте их останавливать:

```bash
docker ps
docker stop CONTAINER_ID
```
